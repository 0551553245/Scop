Manager accounts are created by owner directly. Owner calls supabaseOwner.auth.signUp with manager email password and metadata containing role branch_manager name and branch_id. Then immediately upsert to public.users with id as authData.user.id email role branch_manager name branch_id is_active true using onConflict id ignoreDuplicates false. Then update branches set manager_id to authData.user.id. Show owner modal with email and temporary password. No registration page needed. ID mismatch diagnosis: SELECT a.id as auth_id, u.id as profile_id FROM auth.users a LEFT JOIN public.users u ON u.id = a.id WHERE a.email = the email. If auth_id does not equal profile_id that is the root cause. Safe trigger must have EXCEPTION WHEN OTHERS THEN RETURN new. Logout: branch manager calls supabaseBranchManager.auth.signOut only, owner calls supabaseOwner.auth.signOut only, admin calls supabaseAdmin.auth.signOut only. Never call signOut on all three at once.
